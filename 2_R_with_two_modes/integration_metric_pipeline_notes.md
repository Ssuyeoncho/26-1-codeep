# 통합도 지표와 파이프라인 해석 정리

## 1. 문제의식

이번 분석의 목적은 여러 에이전트 파이프라인에 대해 직접 계산한 `Integration Level`, 즉 통합도와 성능 지표 사이의 관계를 해석하는 것이다.

그래프에서는 통합도를 x축에 두고, 다음 세 가지 성능 지표를 y축에 두었다.

- `Value Conflict Flexibility`
- `Uncertainty Awareness`
- `Multi-dimensional Coverage`

관찰된 전체 경향은 통합도가 높아질수록 성능 지표가 높아지는 방향이지만, 세 지표 모두에서 같은 강도로 나타난 것은 아니다.

- `Uncertainty Awareness`: `r = 0.82, p = 0.023`
  - 통합도와 비교적 강한 양의 상관이 관찰되며, 통계적으로도 유의한 결과로 볼 수 있다.
- `Value Conflict Flexibility`: `r = 0.52, p = 0.236`
  - 양의 경향은 있지만, 표본 수가 작고 p-value가 높으므로 확정적 주장보다는 탐색적 결과로 제시하는 것이 안전하다.
- `Multi-dimensional Coverage`: `r = 0.55, p = 0.199`
  - 마찬가지로 양의 경향은 있으나 통계적으로 강하게 확정하기는 어렵다.

따라서 연구 결과에서는 “통합도가 모든 성능을 일관되게 향상시킨다”라고 단정하기보다, 다음처럼 쓰는 것이 더 적절하다.

> 통합도는 특히 불확실성 인식 능력과 강한 관련을 보였으며, 가치 갈등 유연성과 다차원적 포괄성에 대해서도 양의 경향을 보였다. 다만 후자의 두 지표는 표본 수의 한계로 인해 탐색적 결과로 해석한다.

## 2. 통합도의 핵심 정의

통합도는 단순히 에이전트 수가 많거나, 상호작용 횟수가 많거나, 파이프라인 단계가 길다는 뜻이 아니다.

가장 중요한 구분은 다음이다.

```text
연결성(connectivity):
앞 단계의 output이 뒤 단계의 input으로 전달되는 정도

통합성(integration):
앞 단계 또는 다른 모듈의 산출물이 후속 판단에서 비교, 검증, 수정, 선택, 기억의 근거로 작동하여 최종 판단을 실제로 제약하는 정도
```

즉, 통합도는 단순한 입출력 연결보다 더 강한 개념이다.

```text
전달된다 ≠ 통합된다
```

순차적 파이프라인에서는 앞 단계의 답변이 뒤 단계의 입력으로 들어갈 수 있다. 하지만 그것만으로 통합도가 높다고 보기는 어렵다. 뒤 단계가 앞 단계의 산출물을 검토하거나, 다른 산출물과 비교하거나, 오류를 수정하거나, 최종 선택의 근거로 삼지 않는다면 그것은 단순 전달에 가깝다.

본 연구에서 통합도는 다음처럼 정의할 수 있다.

> 통합도는 에이전트 시스템 내 여러 추론 단계 또는 모듈의 산출물이 단순히 다음 단계로 전달되는 수준을 넘어, 후속 단계에서 비교, 검증, 수정, 선택, 기억의 대상으로 재활용되며 최종 판단을 공동으로 제약하는 정도를 의미한다.

더 짧게 쓰면 다음과 같다.

> 에이전트 시스템이 중간 산출물을 단순 전달하지 않고, 후속 단계에서 평가, 수정, 선택의 근거로 재활용할수록 통합도가 높다.

더 직관적인 표현은 다음과 같다.

> 에이전트 시스템이 여러 답변을 많이 만들수록이 아니라, 만들어진 답변들이 서로를 검토하고 바꾸며 최종 판단에 함께 영향을 줄수록 통합도가 높다.

## 3. 단순 순차성과 통합성의 차이

단순 순차 파이프라인은 다음과 같은 구조를 가질 수 있다.

```text
A가 초안 생성
→ B가 문장 다듬기
→ C가 최종 출력
```

이 경우 A의 산출물이 B의 입력으로 들어가기는 한다. 그러나 B가 A의 판단을 검증하거나, 다른 후보와 비교하거나, 명시적 기준에 따라 수정하거나, C가 A와 B의 차이를 종합하지 않는다면 이는 통합도가 높은 구조라고 보기 어렵다.

반대로 통합도가 높은 구조는 다음과 같다.

```text
A가 주장 생성
B가 반론 생성
C가 A와 B를 비교하고 평가
D가 평가 결과를 반영해 최종 답변 수정
```

여기서는 단순히 산출물이 지나가는 것이 아니라, 각 산출물이 서로 관계를 맺는다.

- A의 주장은 B의 비판 대상이 된다.
- B의 반론은 C의 평가 대상이 된다.
- C의 평가는 D의 수정 방향을 결정한다.
- 최종 답변은 A, B, C의 산출물 간 관계를 반영한다.

따라서 통합도는 파이프라인이 순차적인지 여부가 아니라, 중간 산출물들이 후속 판단을 얼마나 강하게 제약하는지에 의해 결정된다.

## 4. 통합도가 높다는 것은 무엇인가

통합도가 높은 시스템은 다음 특징을 가진다.

- 중간 산출물이 단순히 버려지지 않고 후속 판단의 근거로 사용된다.
- 여러 산출물이 서로 비교된다.
- 하나의 산출물이 다른 산출물의 오류를 드러내거나 보완한다.
- 평가 결과가 최종 답변 수정에 반영된다.
- 이전 단계의 판단이나 상태가 이후 단계에서 기억되어 재사용된다.
- 최종 출력이 단일 모듈의 독립 판단이 아니라 여러 단계의 상호 제약 결과로 형성된다.

반대로 통합도가 낮은 시스템은 다음 특징을 가진다.

- 단일 모델이 한 번에 답을 생성한다.
- 여러 응답이 있더라도 서로 참조하거나 수정하지 않는다.
- 중간 산출물이 단순히 나열되거나 마지막에 표면적으로 집계된다.
- 후속 단계가 이전 단계의 판단을 검증하거나 제약하지 않는다.
- 기억이나 피드백 루프가 없다.

## 5. 각 파이프라인 구성과 차이

### 5.1 Direct LLM

구조:

```text
Input → LLM → Final Answer
```

Direct LLM은 하나의 모델이 입력을 받아 즉시 최종 답변을 생성하는 기본 구조이다. 별도의 검토자, 반론 생성자, 평가자, 기억 모듈, 피드백 루프가 없다.

다른 파이프라인과의 차이:

- 단일 추론 경로만 존재한다.
- 중간 산출물이 외부적으로 분리되어 있지 않다.
- 산출물을 비교하거나 수정하는 후속 단계가 없다.

통합도 해석:

Direct LLM은 구조적으로 통합할 대상이 거의 없기 때문에 가장 낮은 통합도를 가진다. 모델 내부의 추론 과정은 존재하지만, 실험에서 측정하는 파이프라인 수준의 통합 구조는 거의 없다.

### 5.2 Ensemble Vote

구조:

```text
LLM Answer A
LLM Answer B
LLM Answer C
→ Vote / Aggregate
→ Final Answer
```

Ensemble Vote는 여러 개의 독립적인 응답을 생성한 뒤, 다수결 또는 집계 방식으로 최종 답변을 정한다.

다른 파이프라인과의 차이:

- 여러 응답을 생성한다는 점에서 Direct LLM보다 다양성이 크다.
- 하지만 각 응답은 서로 상호작용하지 않는다.
- 응답 A가 응답 B를 비판하거나 수정하지 않는다.
- 마지막 단계는 주로 선택 또는 집계에 가깝다.

통합도 해석:

Ensemble Vote는 병렬적 다양성은 있지만 통합도는 제한적이다. 여러 산출물이 존재하지만, 그 산출물들이 서로를 검증하거나 수정하지 않고 마지막에 표면적으로 합쳐지기 때문이다.

중요한 점:

Ensemble Vote는 “많은 답변을 만든다”는 점에서 복잡해 보일 수 있지만, 이것이 곧 높은 통합도를 의미하지는 않는다. 통합도는 응답 수가 아니라 응답 간 관계와 후속 판단에서의 재사용 정도에 의해 결정된다.

### 5.3 Hierarchical

구조:

```text
Sub-agent 1 → partial judgment
Sub-agent 2 → partial judgment
Sub-agent 3 → partial judgment
        ↓
Supervisor / Higher-level agent
        ↓
Final Answer
```

Hierarchical 구조는 하위 에이전트들이 부분 판단을 수행하고, 상위 에이전트가 이를 종합하여 최종 판단을 내리는 방식이다.

다른 파이프라인과의 차이:

- 역할이 위계적으로 분리되어 있다.
- 하위 산출물이 상위 판단의 입력으로 사용된다.
- 최종 판단은 단일 응답이 아니라 여러 부분 판단의 종합으로 구성된다.

통합도 해석:

Hierarchical 구조는 Direct LLM이나 Ensemble Vote보다 통합도가 높다. 하위 에이전트의 산출물이 상위 에이전트의 판단에 반영되기 때문이다. 다만 상위 에이전트가 하위 산출물을 단순 요약만 하는지, 실제로 비교하고 조정하는지에 따라 통합도는 달라질 수 있다.

### 5.4 Socratic

구조:

```text
Question → Answer
→ Follow-up Question
→ Revised / Deepened Answer
→ Final Answer
```

Socratic 구조는 질문과 답변을 반복하면서 판단 기준을 명시화하고 결론을 유도하는 방식이다.

다른 파이프라인과의 차이:

- 정답을 바로 내기보다 질문을 통해 추론 과정을 단계적으로 전개한다.
- 후속 질문이 이전 답변의 빈틈을 드러내거나 판단 기준을 구체화할 수 있다.
- 여러 독립 에이전트가 병렬로 작동하기보다는 하나의 추론 흐름이 점진적으로 깊어진다.

통합도 해석:

Socratic 구조는 단계 간 의존성이 있으므로 Direct LLM보다 통합도가 높다. 그러나 산출물 간 비교, 반론, 평가, 기억이 강하게 구조화되어 있지 않다면 Debate Judge나 Memory Loop보다 통합도는 낮게 볼 수 있다.

### 5.5 Debate Judge

구조:

```text
Agent A → argument
Agent B → counterargument
Agent C → alternative argument
        ↓
Judge compares and evaluates
        ↓
Final Answer
```

Debate Judge 구조는 여러 에이전트가 서로 다른 주장이나 반론을 제시하고, judge가 이를 비교, 평가하여 최종 판단을 내리는 방식이다.

다른 파이프라인과의 차이:

- 단순히 여러 답을 모으는 것이 아니라, 주장 간 충돌과 비교가 발생한다.
- judge는 각 주장과 반론을 평가 대상으로 삼는다.
- 최종 답변은 특정 주장 하나의 단순 선택이 아니라, 비교와 평가 과정을 거친 결과이다.

통합도 해석:

Debate Judge는 높은 통합도를 가진다. 여러 산출물이 서로를 제약하며, judge의 평가 과정에서 각 산출물이 최종 판단의 근거로 재사용되기 때문이다.

Ensemble Vote와의 차이:

```text
Ensemble Vote:
여러 답변을 독립적으로 만든 뒤 투표한다.

Debate Judge:
여러 주장과 반론을 비교하고 평가한 뒤 판단한다.
```

따라서 Debate Judge는 단순 집계보다 더 강한 통합 구조를 가진다.

### 5.6 Self-Critique

구조:

```text
Draft Answer
→ Critique
→ Revision
→ Final Answer
```

Self-Critique는 모델이 먼저 답변을 생성한 뒤, 그 답변을 스스로 평가하고 수정하는 구조이다.

다른 파이프라인과의 차이:

- 외부 에이전트가 없어도 피드백 루프가 존재한다.
- 초안이 비판 단계의 평가 대상이 된다.
- 비판 결과가 수정 단계의 입력으로 들어간다.
- 최종 답변은 초안과 비판의 결합 결과이다.

통합도 해석:

Self-Critique는 겉으로는 순차적이지만, 단순 순차 파이프라인과 다르다. 두 번째 단계가 첫 번째 산출물을 평가 대상으로 삼고, 세 번째 단계가 그 평가를 반영해 답변을 수정하기 때문이다. 따라서 생성, 평가, 수정이 하나의 피드백 루프로 연결되어 높은 통합도를 가진다.

### 5.7 Memory Loop

구조:

```text
Current input
→ Reasoning / Answer
→ Store state, feedback, or decision
→ Retrieve memory in later step
→ Updated reasoning / Answer
→ Store again
```

Memory Loop는 이전 단계의 판단, 피드백, 상태 정보를 저장하고 이후 추론에 다시 활용하는 구조이다.

다른 파이프라인과의 차이:

- 정보가 한 번 쓰이고 사라지지 않는다.
- 이전 산출물이 이후 판단의 조건이 된다.
- 판단이 시간적으로 누적된다.
- 과거 결정과 현재 판단이 연결된다.

통합도 해석:

Memory Loop는 가장 높은 통합도로 해석할 수 있다. 중간 산출물이 현재 단계에서만 쓰이는 것이 아니라, 이후 단계에서도 반복적으로 재사용되며 시스템의 판단을 지속적으로 제약하기 때문이다.

Self-Critique와의 차이:

```text
Self-Critique:
한 번의 생성-비판-수정 루프가 중심이다.

Memory Loop:
여러 시점의 판단과 상태가 저장되고, 이후 추론에서 반복적으로 재사용된다.
```

따라서 Memory Loop는 시간적 누적성과 재사용성이 더 강하다.

## 6. 파이프라인 간 핵심 차이 요약

| Pipeline | 핵심 구조 | 통합도 판단 기준 | 통합도 해석 |
|---|---|---|---|
| Direct LLM | 단일 입력, 단일 출력 | 비교, 검증, 수정, 기억 없음 | 가장 낮음 |
| Ensemble Vote | 병렬 응답 생성 후 집계 | 응답은 많지만 서로 검증하지 않음 | 낮음-중간 |
| Hierarchical | 하위 판단을 상위 판단이 종합 | 하위 산출물이 상위 판단에 반영됨 | 중간 |
| Socratic | 질문-답변 반복 | 이전 답변이 후속 질문과 추론을 유도 | 중간 |
| Debate Judge | 주장, 반론, judge 평가 | 산출물이 서로 비교·평가됨 | 높음 |
| Self-Critique | 초안, 비판, 수정 | 초안이 평가 대상이 되고 수정에 반영됨 | 높음 |
| Memory Loop | 저장, 검색, 재사용 | 과거 산출물이 이후 판단을 지속적으로 제약 | 가장 높음 |

## 7. 그래프 해석

### 7.1 Value Conflict Flexibility

그래프 정보:

```text
r = 0.52
p = 0.236
```

해석:

통합도와 가치 갈등 유연성 사이에는 양의 경향이 있다. 즉 통합도가 높은 파이프라인일수록 서로 다른 가치나 기준이 충돌하는 상황에서 더 유연하게 대응하는 경향이 나타난다.

그러나 p-value가 높기 때문에 이 결과를 통계적으로 확정적인 관계로 주장하기는 어렵다. 연구 결과에서는 탐색적 경향으로 표현하는 것이 적절하다.

가능한 설명:

통합도가 높은 시스템은 여러 관점, 비판, 기억, 평가를 최종 판단에 반영하기 때문에 단일 관점에 고정될 가능성이 낮다. 따라서 가치 충돌 상황에서 더 유연하게 판단할 가능성이 있다.

보고서용 문장:

> Value Conflict Flexibility에서는 통합도와 양의 상관 경향이 관찰되었으나, 통계적으로 유의한 수준에는 이르지 않았다. 이는 통합 구조가 가치 갈등 상황에서 더 유연한 판단을 가능하게 할 수 있음을 시사하지만, 추가 표본을 통한 검증이 필요하다.

### 7.2 Uncertainty Awareness

그래프 정보:

```text
r = 0.82
p = 0.023
```

해석:

통합도와 불확실성 인식 사이에는 강한 양의 관계가 관찰되었다. 세 그래프 중 가장 설득력 있게 주장할 수 있는 결과이다.

가능한 설명:

통합도가 높은 시스템은 하나의 답변을 바로 확정하기보다, 여러 산출물을 비교하거나, 자기 답변을 비판하거나, 이전 판단을 기억하고 재검토한다. 이러한 구조는 답변의 한계, 불확실한 부분, 판단 보류 가능성을 더 잘 드러내게 한다.

보고서용 문장:

> Uncertainty Awareness에서는 통합도와 강한 양의 상관이 나타났다. 이는 중간 산출물이 평가, 수정, 기억 과정을 통해 재활용되는 시스템일수록 자신의 판단 한계와 불확실성을 더 명시적으로 인식할 가능성이 높음을 시사한다.

### 7.3 Multi-dimensional Coverage

그래프 정보:

```text
r = 0.55
p = 0.199
```

해석:

통합도와 다차원적 포괄성 사이에도 양의 경향이 나타났다. 통합도가 높은 시스템은 여러 관점이나 판단 기준을 결합하기 때문에 더 다양한 차원을 다룰 가능성이 있다.

그러나 p-value가 높으므로 확정적 결과라기보다 탐색적 결과로 보는 것이 적절하다.

가능한 설명:

Debate Judge, Hierarchical, Memory Loop 같은 구조는 서로 다른 역할, 관점, 과거 정보를 통합할 수 있기 때문에 단일 응답보다 더 다양한 측면을 다룰 가능성이 있다.

보고서용 문장:

> Multi-dimensional Coverage에서도 통합도가 높을수록 점수가 상승하는 경향이 관찰되었다. 이는 여러 추론 산출물을 비교하고 재사용하는 구조가 문제를 더 다양한 차원에서 다루는 데 기여할 수 있음을 시사한다. 다만 통계적 유의성은 제한적이므로 후속 검증이 필요하다.

## 8. 연구 결과로 쓸 수 있는 핵심 주장

가장 안전하고 명확한 핵심 주장은 다음과 같다.

> 본 연구에서 통합도는 에이전트 시스템의 단계 수나 상호작용 횟수가 아니라, 중간 산출물이 후속 판단에서 비교, 검증, 수정, 선택, 기억의 대상으로 재활용되며 최종 판단을 공동으로 제약하는 정도를 의미한다.

> 분석 결과, 통합도가 높은 파이프라인일수록 불확실성 인식 능력이 높게 나타났으며, 가치 갈등 유연성과 다차원적 포괄성에서도 양의 경향을 보였다.

> 이는 에이전트 시스템의 성능 향상이 단순히 더 많은 응답을 생성하거나 더 긴 파이프라인을 구성하는 데서 발생하는 것이 아니라, 생성된 중간 산출물들이 후속 판단 과정에서 얼마나 실질적으로 통합되는지와 관련될 수 있음을 시사한다.

## 9. 피해야 할 표현

다음 표현은 피하는 것이 좋다.

```text
에이전트 간 상호작용이 많을수록 통합도가 높다.
```

이 표현이 부족한 이유:

- 상호작용 횟수만 많아도 실제 통합은 약할 수 있다.
- 단순 순차 pipeline도 interaction이 있는 것처럼 보일 수 있다.
- Ensemble Vote처럼 여러 응답이 있어도 서로 검증하거나 수정하지 않으면 통합도가 높다고 보기 어렵다.

다음 표현도 조심해야 한다.

```text
파이프라인이 길수록 통합도가 높다.
```

이 표현이 부족한 이유:

- 단순히 단계가 많아지는 것은 순차성일 뿐이다.
- 통합도는 단계 수가 아니라 산출물 간 관계와 후속 판단에서의 제약력에 의해 결정된다.

## 10. 추천 표현

가장 추천하는 정의 문장:

> 에이전트 시스템이 여러 단계의 산출물을 단순히 다음 단계로 전달하는 데 그치지 않고, 이를 비교, 검증, 수정, 선택, 기억의 대상으로 재활용하여 최종 판단을 공동으로 제약할수록 통합도가 높다.

조금 더 짧은 문장:

> 에이전트 시스템이 중간 산출물을 단순 전달하지 않고, 후속 단계에서 평가, 수정, 선택의 근거로 재활용할수록 통합도가 높다.

직관적인 설명:

> 통합도는 여러 답변을 많이 만드는 정도가 아니라, 만들어진 답변들이 서로를 검토하고 바꾸며 최종 판단에 함께 영향을 미치는 정도이다.

논문식 표현:

> Integration Level captures the extent to which intermediate outputs in an agentic pipeline are not merely passed forward, but are reused as objects of comparison, verification, revision, selection, or memory, thereby jointly constraining the final decision.

한국어 번역:

> 통합도는 에이전트 파이프라인의 중간 산출물이 단순히 다음 단계로 전달되는 것이 아니라, 비교, 검증, 수정, 선택, 기억의 대상으로 재사용되어 최종 판단을 공동으로 제약하는 정도를 포착한다.

## 11. 한 문단 요약

본 연구에서 통합도는 파이프라인의 길이나 단순 상호작용 횟수가 아니라, 중간 산출물들이 후속 판단 과정에서 얼마나 실질적으로 재활용되고 서로를 제약하는지를 나타내는 지표이다. Direct LLM은 단일 응답 구조이므로 통합도가 낮고, Ensemble Vote는 여러 응답을 생성하지만 서로를 검증하거나 수정하지 않기 때문에 통합도는 제한적이다. 반면 Debate Judge, Self-Critique, Memory Loop는 주장, 비판, 평가, 수정, 기억을 통해 중간 산출물이 최종 판단에 구조적으로 반영되므로 높은 통합도를 가진다. 그래프 분석 결과, 통합도는 특히 불확실성 인식과 강한 양의 상관을 보였으며, 가치 갈등 유연성과 다차원적 포괄성에서도 양의 경향을 보였다. 이는 에이전트 시스템의 성능이 단순히 더 많은 응답이나 더 긴 절차가 아니라, 중간 산출물들이 후속 판단을 얼마나 효과적으로 제약하고 통합하는지와 관련될 수 있음을 시사한다.
